A Meaningful Messaging Bot

This is a simple bot for a party supplies store. It has two functions: looking up a customer's order and answering general questions.

The order lookup asks for a four-digit order number, then returns the customer name, product ordered, and order status (e.g., shipped or pending). For the purposes of this demo, there is mock data for order numbers 1001–1005.

The actual lookup in the bot is handled via a FaaS integration that passes the order number as a bot variable. The FaaS function contains mock JSON data to emulate an API lookup; it either matches the provided order number or returns an "order not found" message. It then formats the JSON data into a readable string and sends it back to the bot.

The second function of the bot is to answer questions. For simplicity, this version provides literal answers to three different topics using an inline bot flow. In reality, this would likely be an intent-based KB or LLM task.
