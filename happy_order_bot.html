A Meaningful Messaging Bot

This is a simple bot for a party supplies store. It has two functions: look up a customer's order and ask general questions.

The order lookup ask for a four digit order number then returns customer name, product ordered and status of order (shipped, pending, etc). For the purposes of this demo, there is mock data for order number using the numbers 1001-1005.

The actual lookup in the bot is done with a FaaS integration that passes the order number provided as a bot variable. The FaaS fucntion contains mock Json data to emulate an API lookup, and it matches the order number provided or returns an order not found message. It then formats the Json data into a readable string and sends it back to the bot.

The second function of the bot is to answer questions. For simplicity, this version just provides literal questions to three different topics with an inline bot flow. In reality this would likely be an intent based KB or LLM task.
